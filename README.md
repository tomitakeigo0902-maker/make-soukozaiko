# 倉庫在庫管理アプリ

工場倉庫の原料の入庫・出庫を記録し、現在庫を自動で計算する社内向け Web アプリです。
発注点を下回った原料は画面上でアラート表示します。

- 同じ部署の数人でブラウザから共有して使えます
- 社内サーバーに **exe を 1 ファイル置くだけ**（各 PC へのインストール不要）
- データは `inventory.db`（SQLite）1 ファイルに保存（バックアップはコピーするだけ）

## 主な機能

| 画面 | できること |
|------|-----------|
| 在庫一覧 | 原料ごとの現在庫を保管場所別に表示。発注点割れの原料を赤くハイライト＋件数アラート |
| 入出庫登録 | 入庫・出庫（生産ラインへの払い出し）を1件ずつ記録。入庫記録（原料用・包材用）の Excel(.xlsx) からの一括取り込みにも対応 |
| 一括出庫 | 出庫依頼表のように複数の原料をまとめて出庫登録 |
| 履歴 | 入出庫の記録を一覧・編集・削除。原料で絞り込み |
| 原料マスター | 原料一覧の Excel(.xlsx) を「カタログ」に取り込み、その中から実際に在庫する原料を保管場所ごとに選んで登録。手入力での登録・編集も可能 |

- 現在庫を超える出庫は登録できません（1件ずつの登録ではエラー、一括出庫では在庫が足りる明細だけ登録し、不足分は知らせます）
- 発注点を `0` にした原料はアラート対象外になります

## 技術構成

- バックエンド: Python 3.11 + FastAPI
- データベース: SQLite（`inventory.db`）
- フロントエンド: 素の HTML + JavaScript（フレームワークなし）
- 配布: PyInstaller で単一 exe 化

```
make-soukozaiko/
├── main.py            アプリ本体・API・起動処理
├── database.py        SQLite の接続とテーブル定義
├── static/index.html  画面（1 ファイル）
├── requirements.txt   必要なライブラリ
├── build.bat          exe をビルドする（Windows）
└── run_dev.bat        ソースのまま起動して動作確認する（Windows）
```

## 使い方

### 事前準備：Python のインストール（手元の PC）

`run_dev.bat`（動作確認）と `build.bat`（exe 作成）は Python を使います。
まだ入っていない場合は、先にインストールしてください。

1. https://www.python.org/downloads/ から Python 3.11 以上をダウンロード
2. インストーラを実行し、**最初の画面で「Add python.exe to PATH」に必ずチェックを入れる**
3. インストール後、コマンドプロンプトで `python --version` と打ち、
   バージョンが表示されれば成功

> ※ 社内サーバーには Python のインストールは不要です（exe で動くため）。
> 必要なのは手元の開発用 PC だけです。

### A. 開発・動作確認（手元の PC）

上記の Python が入った PC で動作を確認できます。

Windows の場合は `run_dev.bat` をダブルクリック。
コマンドで実行する場合は次のとおりです。

```bash
pip install -r requirements.txt
python main.py
```

起動したらブラウザで `http://localhost:8000/` を開きます。

### B. exe を作る（配布用）

Python が入った PC で `build.bat` をダブルクリックします。
`dist\soukozaiko.exe` が生成されます。

> exe のビルドは Windows 上で行ってください（Windows 用 exe は Windows でしか作れません）。

### C. 社内サーバーへの配置

1. できあがった `soukozaiko.exe` を社内サーバーの任意のフォルダにコピーする
2. サーバー上で `soukozaiko.exe` をダブルクリックして起動する
   （Python のインストールは不要です）
3. 各 PC のブラウザから `http://サーバー名:8000/` にアクセスする
   - 起動時にコンソールへアクセス用 URL が表示されます
   - よく使う PC ではこの URL をブックマークしておくと便利です
4. 同じフォルダに `inventory.db` が作られ、ここに全データが保存されます

### バックアップ

`inventory.db` をコピーするだけでバックアップ完了です。
定期的に別の場所へコピーする運用を推奨します。

### 補足

- ポート `8000` が他のソフトと競合する場合は `main.py` の `PORT` を変更してください
- 各 PC からアクセスできない場合は、サーバーの Windows ファイアウォールで
  ポート `8000` の受信を許可してください
